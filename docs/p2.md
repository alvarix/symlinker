# New use case
This app symlinks folders into obsidian so I can read project documentation while leaving the files in the projects. 

I decided to add folders so I could add a whole project root and all it's docs would be exposed in a file tree with the project as the root, therefore ensuring each doc had the context of it's project in obsidian. 

I have a new use case - I want to symlink an obsidian note into a project so an LLM can read my notes. 
When I tried to select source it would only allow my to add a folder, not a file. 

On the surface it seems that file selection should be re-enabled to allow this use case. Since it touches the core of the app a full analysis should be done. 

If that can be done without any implications then proceed, updating readme.
